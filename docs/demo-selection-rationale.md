# Demo selection rationale

## Miksi juuri nämä demot?

Valitut demot muodostavat yhdessä tarinan siitä, miten epäselvästä mahdollisuudesta syntyy käytännön työväline.

AI-mahdollisuus → tarkistusmalli → työjono → raportti → priorisoitu toimenpide

## 1. AIAuditLab

AIAuditLab on valittu päädemoksi, koska se sopii parhaiten AI-avusteisen markkinoinnin ja sisältötyön kulmaan.

Se näyttää, miten AI:hin liittyvä epäselvä kysymys voidaan muuttaa hallituksi tarkistusprosessiksi.

### Mitä tämä todistaa

- osaan rajata AI-käyttötapauksia
- ymmärrän ihmisen ja AI:n roolien eron
- osaan rakentaa paikallisen tarkistusprosessin
- osaan tehdä prompt-pohjaisesta työstä hallittavaa
- ymmärrän AI:n käytännön käyttöönoton riskit

## 2. Content Ops Platform

Content Ops Platform on valittu, koska se näyttää kyvyn rakentaa päätöstyötila laajalle sisältö- ja sivumassalle.

Se ei ole pelkkä raportti, vaan työväline, jonka tarkoitus on auttaa ihmistä päättämään mitä tehdään seuraavaksi.

### Mitä tämä todistaa

- osaan jäsentää hajanaisia lähdeaineistoja
- osaan suunnitella tarkistusjonoja
- ymmärrän sisältöjen, SEO:n ja sivurakenteen yhteyden
- osaan erottaa havainnot, suositukset ja päätökset toisistaan
- osaan suunnitella turvallisia read-only- ja dry-run-työnkulkuja

## 3. SearchAudit

SearchAudit on valittu, koska se näyttää SEO:n, sisäisen haun ja termikandidaattien muuttamisen arvioitavaksi työjonoksi.

Se yhdistää hakukäyttäytymisen, termit, kohdesivun, tarkistustilat ja vientitiedostot.

### Mitä tämä todistaa

- osaan muuttaa hakudatan käytännön toimenpiteiksi
- osaan suunnitella arviointi- ja hyväksyntätyönkulkuja
- ymmärrän sisäisen haun ja hakukäyttäytymisen yhteyden sisältökehitykseen
- osaan rakentaa paikallisia SEO-työkaluja
- osaan tehdä tarkistuspaketteja jatkotyötä varten

## 4. Piwik PRO Workspace

Piwik PRO Workspace on tukidemo, joka näyttää analytiikkadatan muuttamista priorisoiduiksi käyttäjäpolkuhavainnoiksi.

Se täydentää SEO- ja Content Ops -kulmaa näyttämällä, että ymmärrän myös käyttäjäpolkujen ja konversiopolkujen analytiikkaa.

### Mitä tämä todistaa

- osaan ajatella analytiikkaa operatiivisena työnkulkuna
- ymmärrän käyttäjäpolkujen pudotuskohtia
- osaan suunnitella diagnoosikortteja ja prioriteettitaulukoita
- osaan yhdistää analytiikan ja kehitystoimet

## Kokonaisuus

Yhdessä demot näyttävät tämän osaamisketjun:

1. tunnistan epäselvän ongelman
2. teen siitä tarkistettavan kokonaisuuden
3. rakennan työnkulun
4. muutan havainnot toimenpiteiksi
5. teen lopputuloksesta käytettävän arjen työssä
