# Portfolio Demos – Reko Mäkelä

Tämä repo kokoaa demoja, malleja ja työnkulkuja, jotka näyttävät miten muutan markkinoinnin, SEO:n, sisällön, analytiikan ja AI:n mahdollisuuksia käytännön tekemiseksi.

Painopiste ei ole yksittäisessä teknologiassa, vaan siinä miten epäselvä tarve muutetaan selkeäksi etenemiseksi:

selvitys → analyysi → priorisointi → työnkulku → toteutus → mittaaminen → jatkokehitys

## Päädemot

### 1. AIAuditLab

Epämääräisestä AI-mahdollisuudesta hallituksi tarkistus- ja työnkulkumalliksi.

Lisätiedot: demos/01-ai-audit-lab/README.md

### 2. Content Ops Platform

Hajanaisista sisältö- ja sivuhavainnoista päätöstyötilaksi.

Lisätiedot: demos/02-content-ops-platform/README.md

### 3. SearchAudit

Hakudatasta ja termikandidaateista arvioitavaksi kehitysjonoksi.

Lisätiedot: demos/03-search-audit/README.md

### 4. Piwik PRO Workspace

Analytiikkadatasta priorisoiduiksi käyttäjäpolkuhavainnoiksi.

Lisätiedot: demos/04-piwik-pro-workspace/README.md

## Mitä tämä portfolio näyttää

- AI-avusteisten markkinointi- ja sisältötyönkulkujen suunnittelua
- hakudatan, analytiikan ja käyttäjäpolkujen muuttamista toimenpiteiksi
- laajojen sisältö- ja sivumassojen jäsentämistä tarkistusjonoiksi
- paikallisesti toimivien työkalujen ja raporttien suunnittelua
- kykyä erottaa automaatio, ihmisen päätös ja tarkistusvaiheet toisistaan

## Ydinviesti

En tee vain raportteja tai yksittäisiä kokeiluja. Rakennan työnkulkuja, joissa data, sisältö, AI ja analytiikka muuttuvat selkeiksi päätöksiksi ja seuraaviksi toimenpiteiksi.

## Rajaukset

- Demot eivät sisällä oikeaa asiakasdataa julkisessa muodossa.
- Salaisuuksia, API-avaimia tai palvelutilitiedostoja ei tule julkaista.
- Osa demoista on paikallisia kehitystyökaluja, ei tuotantojärjestelmiä.
- AI toimii työn valmistelun ja jäsentämisen tukena; ihminen säilyy päätöksentekijänä.

## Seuraavat parannukset

Tämän koosterepon seuraavat vaiheet:

1. Lisätä jokaiselle demolle yksi kuvakaappaus tai lyhyt GIF.
2. Lisätä jokaiselle demolle “mitä tämä todistaa osaamisestani” -osio.
3. Tehdä `docs/demo-selection-rationale.md`, jossa perustellaan miksi juuri nämä demot nostetaan portfolioon.
4. Lisätä public-safe huomio: demot eivät sisällä asiakasdataa, salaisuuksia tai tuotantojärjestelmien tietoja.
5. Linkittää varsinaisiin demo-repoihin, kun niiden README:t on siistitty.
